# Stage 6 of the pipeline reads this file and removes any matching patient
# (and all their records) from the final dashboard bundle.
#
# One rule per line. Lines starting with # are comments. Blank lines are ignored.
#
# Four match modes:
#   id: <patient_id>           -> exact patient_id (UUID)
#   name: <First Last>         -> exact full name match (case-insensitive)
#   name_contains: <substring> -> first or last name contains (case-insensitive)
#   mrn_contains: <substring>  -> MRN contains (case-insensitive)
#
# Lines without a "type:" prefix default to name_contains.
